Codility.com

I'm looking for a job at the moment and recently had a test on Codility.com. I've heard of it before but never had the opportunity to use it for real so I'll take this opportunity to write about it.

Basically it’s an automated way of vetting people prior to an interview. It’s pretty well known in software that a lot of programmers can’t program. I've seen this myself when I was (briefly) involved in hiring a developer for a previous company.

I like to think of myself as somebody who can, in fact, program, however I did find the challenges, at least with the pressure of a time limit, fairly challenging. I think it’s a very good idea and I did quite enjoy taking part although in this case I think nerves got the better of me and I didn't represent myself as well as I could have.

The main problem I have with it is that when I write some code that looks like it should work, and it doesn't - it doesn't inform me of the inputs that caused it to fail. This is especially clear during the current certification task  where my latest effort is:

def shortest_adj_seq(A):

if len(A) == 0: return 0

positions = {}

positions[A[len(A) - 1]] = 1

for i in range(len(A) - 2, -1, -1):

if i > 0 and (A[i - 1]) in positions:

l = positions[A[i-1]]

else:

l = 9999

r = positions[A[i+1]]

if l < r:

new = l + 1

else:

new = r + 1

if A[i] not in positions or new < positions[A[i]]:

positions[A[i]] = new

return positions[A[0]]

It works on the test cases I've created for it but I guess I fail at creating robust test cases. Can anyone point out what is causing this to fail?

All in all this is a lot more fun than phone interviews asking about HTML standards and I imagine it offers more information for the company (as long as they look at the code and don’t blindly follow the percentages). I definitely hope to come across it in future jobs I apply for.
